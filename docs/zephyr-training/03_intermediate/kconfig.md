# Kconfig

Kconfig is the configuration system Zephyr inherited from the Linux kernel. It controls which drivers, subsystems, and features are compiled into your firmware.

Every Zephyr feature — I2C, BLE, USB, logging — is gated behind a `CONFIG_` symbol. If `CONFIG_I2C=n`, the I2C driver is not compiled and takes zero flash space.

<br/>

---

## Your project's prj.conf

```kconfig title="prj.conf"
# Enable the LED strip driver (for WS2812)
CONFIG_LED_STRIP=y

# Logging
CONFIG_LOG=y
CONFIG_LOG_DEFAULT_LEVEL=3   # 0=off  1=err  2=warn  3=info  4=debug
```


<br/>

---

## How are Kconfig values resolved?

Zephyr merges multiple Kconfig files in order:

1. **SoC defaults** — from the chip definition (`esp32s3.conf`)
2. **Board defaults** — from the board definition (`esp32s3_devkitc.conf`)
3. **Your `prj.conf`** — your application overrides
4. **Driver Kconfig** — for every symbol you enable (e.g. `CONFIG_LED_STRIP=y`), the corresponding driver is compiled into your firmware. The overlay describes the hardware — Kconfig makes the driver exist.

Values in `prj.conf` override everything upstream. If you set `CONFIG_LOG_DEFAULT_LEVEL=4`, it overrides the board default.

:::info
After a build, `cat build/zephyr/.config` shows every resolved Kconfig value — the definitive record of what is compiled into your firmware.
:::

## Dependency-driven Kconfig

Some symbols have `depends on` — if a driver requires another subsystem, you must enable the full chain or the build will fail.

To understand what a driver requires, read its `Kconfig` file directly. For the WS2812 LED strip at `zephyr/drivers/led_strip/Kconfig`:

```kconfig
config WS2812_STRIP_I2S
    bool "WS2812 LED strip I2S driver"
    default y
    depends on DT_HAS_WORLDSEMI_WS2812_I2S_ENABLED
    select I2S if $(dt_compat_on_bus,$(DT_COMPAT_WORLDSEMI_WS2812_I2S),i2s)
    help
      Enable driver for WS2812 (and compatible) LED strips using I2S.
```

Three things to notice:

- **`default y`** — `WS2812_STRIP_I2S` is auto-enabled. You do not set it manually.
- **`depends on DT_HAS_WORLDSEMI_WS2812_I2S_ENABLED`** — this symbol is auto-generated by Zephyr when it finds a DTS node with `compatible = "worldsemi,ws2812-i2s"` in your overlay. The overlay triggers the Kconfig.
- **`select I2S`** — the driver automatically pulls in the I2S subsystem. You do not need `CONFIG_I2S=y` in your `prj.conf`.

This is why `CONFIG_LED_STRIP=y` alone is enough — everything else resolves automatically from your overlay.

## Common Kconfig types

| Symbol type | Example | Meaning |
|---|---|---|
| `bool` | `CONFIG_LED_STRIP=y` | Enable/disable a feature |
| `int` | `CONFIG_SAMPLE_LED_BRIGHTNESS=16` | Numeric value |
| `string` | `CONFIG_BOOT_BANNER_STRING="my fw"` | String value |
| `hex` | `CONFIG_SRAM_SIZE=0x40000` | Hex value |

## Browsing available symbols

```bash
# Launch the Kconfig GUI (requires Python and tkinter)
west build -t menuconfig

# Or the simpler ncurses version
west build -t guiconfig
```

Both show the full tree of available symbols with their current values and help text.
